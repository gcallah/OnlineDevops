1. Microservices: Tiny applications that do a single job, like authentication or scheduling deliveries.
  2. Kanban: A tool for controlling work-in-progress and making visible who is working on what tasks.
  3. Monitoring: Software that "watches" an application and sends out alerts at signs of trouble.
  4. Kubernetes: An open-source tool for orchestrating containers.
  5. Agile Development: A method of developing software featuring frequent releases, adaptability to change, and close collaboration with customers.
  6. make: A venerable tool for automating builds.
  7. Test-Driven Development: A method of developing software that writes tests for a new feature first, then writes the feature.
  8. Continuous Deployment (CD): Regularly deploying new work into production, usually with every push to the master branch on the origin source-code control server.
  9. Containerization: The process of packaging an application and everything it depends upon in a "container," then running it in its own virtual environment.
  10. Docker: The most popular tool for containerizing applications.
  11. Version Control: A tool that allows users to keep multiple versions of a file, revert to earlier versions, compare versions, and so on.
  12. Slack: A popular tool for communication in development teams; also, the "empty" periods that are necessary in a schedule for anyone to do creative work.
  13. Continuous Integration (CI): The process of integrating different developers' work with high regularity.
